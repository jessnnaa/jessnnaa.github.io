02 — Problem

무엇이 문제였는가

스팀과 같은 기존 플랫폼은 상점과 신작 홍보를 전면에 배치합니다.
유저가 런처를 열 때 원하는 것은 게임을 하는 것인데, 플랫폼은 결제를 유도하는 경향이 강합니다.

비효율적인 탐색

설치된 게임 수가 늘어날수록 원하는 게임을 찾는 데 시간이 걸립니다. 최근 플레이나 즐겨찾기에 대한 접근성이 낮아 불필요한 스크롤과 탐색이 반복됩니다.

정보의 피로도

UI가 정보 위주로 구성되어 시각적 피로도가 높습니다. 프로모션·신작·추천 콘텐츠가 혼재하면서 유저의 주의를 분산시킵니다.

불명확한 상태

설치 여부, 업데이트 필요, 실행 가능 등 게임 상태가 직관적으로 전달되지 않습니다. 유저는 매번 확인하는 과정을 거쳐야 합니다.

03 — Target User

누구를 위한 설계인가

다양한 장르의 게임을 동시에 설치해두고, 그때그때 원하는 게임에 빠르게 접근하고자 하는 유저. 탐색 시간이 길어질수록 플레이 의욕이 낮아집니다.

▶

빠른 접근 욕구

런처를 열면 곧바로 게임을 시작하고 싶어합니다. 탐색 과정이 길어지면 몰입 전 피로가 쌓입니다.

≡

멀티 라이브러리

10개 이상의 게임을 설치해두고 상황에 따라 전환하며 플레이합니다. 정리된 목록과 빠른 필터가 핵심입니다.

◉

즉각적 피드백 기대

버튼 클릭에 즉각 반응이 없으면 오류로 인식합니다. 게임 유저는 UI 반응 속도에 민감합니다.

04 — Design Direction

UX 개선 방향

방향 01

탐색 시간 단축

최근 플레이한 게임을 홈 최상단에 배치하고, 즐겨찾기하고자 하는 게임을 고정할 수 있는 핀 기능을 도입합니다. 자주 실행하는 게임에 불필요한 탐색 없이 즉시 접근할 수 있도록 합니다.

방향 02

정보 구조 단순화

카테고리를 재정리하고 불필요한 프로모션 요소를 제거합니다. 상점보다 라이브러리를 우선하는 구조로, 유저가 게임 자체에 집중할 수 있는 화면을 만듭니다.

방향 03

상태 가시성 강화

실행 가능 / 설치 필요 / 업데이트 필요 상태를 버튼 레벨에서 명확히 구분합니다. 유저가 별도로 확인하지 않아도 다음 행동을 바로 취할 수 있도록 돕습니다.

05 — User Flow

실행까지의 흐름

핵심 플로우는 최대한 단순하게, 불필요한 중간 단계를 제거하고 홈에서 실행까지 최단 경로를 설계하였습니다.

Home
→

→
상세 화면

실행
개선 포인트 — 홈에서 최근 플레이 게임을 바로 실행할 수 있어 상세 화면을 거치지 않아도 됩니다.
실행까지의 최소 단계가 Home → 실행으로 단축됩니다.

07 — UI Design

UI 디자인

게임 UX의 핵심은 즉각적인 피드백과 명확한 상태 표현입니다.

Button State

버튼 상태 구분

실행 가능 / 설치 필요 / 업데이트 필요를 색상과 레이블로 명확히 구분합니다. 유저는 버튼 하나만 봐도 다음 행동을 알 수 있어야 합니다. 모호하게 둬서는 안 됩니다.

Visual Hierarchy

시선 흐름 설계

가장 중요한 액션 버튼을 가장 크고 높은 대비로 배치합니다. 유저의 시선이 자연스럽게 핵심 행동으로 유도되도록 계층을 명확히 설계했습니다.

Feedback

즉각적 피드백

클릭 즉시 상태 변화를 시각적으로 전달합니다. 게임 유저는 반응이 없는 UI를 오류로 인식합니다. 피드백의 명확성이 사용자 경험의 질을 높이는 데 결정적입니다.

08 — Outcome

결과

탐색 시간 감소


최근 플레이 및 핀 기능으로 원하는 게임에 더 빠르게 도달할 수 있습니다.

실행 단계 단순화


홈에서 직접 실행이 가능해 불필요한 중간 화면을 거치지 않아도 됩니다.

사용자 피로도 감소


정보 구조 단순화와 명확한 상태 표현으로 인지 부담이 줄어들었습니다.

이 프로젝트는 상점 중심 구조에서 플레이 중심 구조로의 전환을 핵심으로 합니다.
게임 런처는 게임을 파는 공간이 아니라, 게임을 즐기는 공간이어야 합니다.
유저 친화적 플랫폼은 유저가 원하는 것을 가장 빠르게 전달하는 데서 시작합니다.
